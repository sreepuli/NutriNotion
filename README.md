A personalized nutrition tracking app powered by AI. NutriNotion generates daily meal plans tailored to each user's health goals, dietary preferences, and available mess/cafeteria menu — then tracks calorie intake as meals are checked off throughout the day.

---

## Features

- **AI-Powered Meal Planning** — Daily personalized meal plans (breakfast, lunch, snacks, dinner) generated by Google Gemini AI
- **Mess Menu Integration** — Pulls from the available weekly cafeteria menu so suggestions are practical
- **Calorie Tracking** — Real-time consumed vs. target vs. remaining calories updated as you check off meals
- **Smart Onboarding** — Collects height, weight, age, dietary preferences, allergies, activity level, and health goals to compute personalized calorie and macro targets
- **Health Goal Support** — Weight loss, maintenance, or muscle gain goals
- **Progress Analytics** — Daily, weekly, and monthly nutrition insights with streak tracking
- **Time-Aware UI** — Meal cards adapt based on current time (past/current/upcoming)

## API Overview

| Endpoint                                               | Description                                    |
| ------------------------------------------------------ | ---------------------------------------------- |
| `POST /api/auth/signup`                                | Register a new user                            |
| `POST /api/auth/login`                                 | Login and receive auth token                   |
| `PUT /api/onboarding/{userId}`                         | Save onboarding profile, returns macro targets |
| `GET /api/onboarding/{userId}`                         | Fetch saved profile                            |
| `POST /api/personalized-meals/{userId}/generate-today` | Generate today's AI meal plan                  |
| `GET /api/personalized-meals/{userId}/today`           | Fetch today's meal plan                        |
| `PUT /api/personalized-meals/item/{itemId}/check`      | Check/uncheck a meal item                      |
| `GET /api/calories/{userId}/today`                     | Get today's calorie summary                    |
| `GET /api/menu/{day}`                                  | Fetch mess menu for a specific day             |
| `GET /api/menu`                                        | Fetch full weekly mess menu                    |

## Getting Started

### Prerequisites

- Java 21+
- Maven 3.9+
- PostgreSQL 14+
- Flutter 3.6.2+ / Dart 3.6.2+

---

### Backend Setup

1. **Create the PostgreSQL database:**

   ```bash
   createdb nutrinotion
   ```

2. **Configure credentials** — edit `NutriNotion-backend/src/main/resources/application.properties`:

   ```properties
   spring.datasource.url=jdbc:postgresql://localhost:5432/nutrinotion
   spring.datasource.username=<your_pg_username>
   spring.datasource.password=<your_pg_password>

   gemini.api.key=<your_gemini_api_key>
   ```

3. **Build and run:**

   ```bash
   cd NutriNotion-backend
   mvn clean package
   mvn spring-boot:run
   ```

   The API will be available at `http://localhost:8080`.

---

### Frontend Setup

1. **Install dependencies:**

   ```bash
   cd NutriNotion-Frontend
   flutter pub get
   ```

2. **Configure the base URL** in `lib/services/api_client.dart`:
   - **Android Emulator** — default `http://10.0.2.2:8080/api` works out of the box
   - **Physical Android Device** — run `adb reverse tcp:8080 tcp:8080`, then update `androidBaseUrl` to `http://127.0.0.1:8080/api`
   - **Web / Desktop** — uses `http://localhost:8080/api` by default

3. **Run the app:**
   ```bash
   flutter run
   ```
